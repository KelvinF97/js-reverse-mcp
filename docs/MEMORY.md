# JS Reverse MCP - Memory System

## 简介

记忆系统允许你将逆向分析过程中的重要发现（例如加密函数位置、API 签名算法、关键参数）保存到本地，以便下次启动时直接调用，而无需重新抓包分析。

目前版本使用本地文件 `.mcp_memory.json` 进行持久化存储。

## 工具列表

### `remember_insight`

保存一条新的知识点。

**参数：**
- `key`: 唯一的键名（例如 `xhs_sign_func`）。
- `value`: 知识内容（例如 `Encryption function is located at app.js line 123`）。
- `tags`: (可选) 标签数组（例如 `["encryption", "sign"]`）。

**示例：**
```
remember_insight(
  key="xhs_sign_v1",
  value="The x-s signature is generated by window._webmsxyw(url, data)",
  tags=["xhs", "sign"]
)
```

### `recall_insight`

根据键名或关键词搜索记忆。

**参数：**
- `query`: 要搜索的键名或关键词。

**示例：**
```
recall_insight(query="xhs_sign")
```

### `list_insights`

列出所有保存的记忆。

## 最佳实践

1.  **分析时随手记**：每当你通过 `search_in_sources` 或 `break_on_xhr` 找到关键代码位置时，立即调用 `remember_insight`。
2.  **结构化命名**：建议使用 `项目名_模块名_功能` 的命名方式（如 `tiktok_login_captcha`）。
3.  **启动时检查**：每次开始新的 Session，先运行 `list_insights` 看看之前积累了什么。
